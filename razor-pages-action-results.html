Razor 页面中的返回结果

Razor 页面中的操作结果通常用作Handler 方法 的返回类型，并负责生成响应和适当的状态代码。 操作结果实现抽象的 Microsoft.AspNetCore.Mvc.ActionResult 类， 或 Microsoft.AspNetCore.Mvc.IActionResult 接口。 ASP.NET Core 包含三个以上的 ActionResult 类，涵盖了广泛的需求， 包括但不限于执行并返回Razor页面的内容，返回文件的内容， 重定向到另一个资源或简单地返回特定的 HTTP 状态代码。 如果您需要定制的东西，也可以创建自己的 ActionResult 类。

下面的代码说明了 RedirectToPageResult 类在一个简单的 OnGet 处理程序方法中的使用，这导致用户被重定向到指定的页面：

  1. public IActionResult OnGet()
  2. {
  3.     return new RedirectToPageResult("Index");
  4. }

OnGet handler 方法的返回类型为 IActionResult ， 这意味着返回类型可以是在其继承层次结构中某处实现 IActionResult 接口的任何类。 最终，这表示着代表整个框架中的任何返回结果。 通常，handler 方法返回类型尽可能具体是一个好方式，因此应按如下方式重构此示例： 

  1. public RedirectToPageResult OnGet()
  2. {
  3.     return new RedirectToPageResult("Index");
  4. }

如果您的处理程序方法根据条件返回不止一种类型的操作结果，则需要将方法的返回类型扩展为 ActionResult 或 IActionResult， 如本示例所示，如果 ModelState 无效，则返回 PageResult 或 RedirectToPageResult （不同的类型） 如果 ModelState 有效： 

  1. public IActionResult OnPost()
  2. {
  3.     if(!ModelState.IsValid)
  4.     {
  5.         return new PageResult();
  6.     }
  7.     // otherwise do some processing
  8.     return new RedirectToPageResult("Index");
  9. }

许多操作结果都在 Razor 页面 PageModel 类上定义了关联的辅助方法，这些方法不需要“新建”ActionResult 类的实例，从而有助于简化代码。 Page() 方法返回 PageResult， RedirectToPage 方法返回 RedirectToPageResult，因此可以使用这些方法简化前面的示例： 

  1. public IActionResult OnPost()
  2. {
  3.     if(!ModelState.IsValid)
  4.     {
  5.         return Page();
  6.     }
  7.     // otherwise do some processing
  8.     return RedirectToPage("Index");
  9. }

注释

  1. 重定向操作结果包括用于指定重定向应该是永久的和/或 HTTP 动词应该跨重定向保留的选项。 临时重定向由 HTTP 状态代码 302 和 307 表示。永久重定向由状态代码 301 和 308 表示。 永久重定向指令通常受到搜索引擎和其他索引技术的支持，它们将用新位置替换其索引中的现有条目，在操作结果生成的位置标头中指定。

    如果您不选择保留原始请求使用的 HTTP 方法，则重定向的请求可能（并且经常会）使用 GET，而不管原始请求使用的方法如何。 如果要确保为后续请求（以及可能的任何表单值）保留原始 HTTP 方法，则应使用名称中包含 PreserveMethod 的选项之一， 或者为更通用的 RedirectResult 中的适当参数指定 true 方法。

  2. 传递给 RedirectToPageResult 方法的值是相对于当前页面的， 除非它以正斜杠 (/) 为前缀， 在这种情况下它是相对于 Pages 文件夹的。

  3. Partial 辅助方法从 ASP.NET Core 2.2 开始可用。
