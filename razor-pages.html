Razor 页面文件

Razor 页面文件

所有 Razor 文件都以 .cshtml 结尾。大多数 Razor 文件都是可浏览的，并且包含客户端和服务器端代码的混合，在处理时，会将 HTML 发送到浏览器。 这些页面通常称为“内容页面”。 本节更深入地了解内容页面及其相关的 PageModel 文件。

内容页面

要使文件充当 Razor 内容页面，它必须具有三个特征：

  • 它的文件名中不能是首字母下划线
  • 文件扩展名是 .cshtml
  • 文件的第一行是 @page

将 @page 指令作为第一行代码是至关重要的。 如果不这样做，该文件将不会被视为 Razor 页面，并且如果您尝试浏览它也不会被找到。 @page 指令前可以有空格，但不能有任何其他字符，即使是空代码块。 与 @page 指令在同一行上允许的唯一其他内容是 route 模板。

内容页面可以指定布局文件，但这不是强制性的。 它们可以选择包含代码块、HTML、JavaScript 和内联 Razor 代码。

Razor 语法

内容页面主要由 HTML 组成，但它们也包括 Razor 语法 内容中的可执行 C# 代码。 C# 代码在服务器上执行，通常会导致发送到浏览器的响应中包含动态内容。

单文件方法

尽管不推荐，但可以开发仅依赖于内容页面的 Razor Page 应用程序。 以下示例提供了一种与具有脚本编写背景的开发人员（如 PHP 或经典 ASP）所熟悉的方法最相似的方法：

Example.cshtml:

@page 
@{
    var name = string.Empty;
    if (Request.HasFormContentType)
    {
        name = Request.Form["name"];
    }
}

<div style="margin-top:30px;">
    <form method="post">
        <div>Name: <input name="name" /></div>
        <div><input type="submit" /></div>
    </form>
</div>
<div>
    @if (!string.IsNullOrEmpty(name))
    {
        <p>Hello @name!</p>
    }
</div>

Razor 内容页面需要文件顶部的 @page 指令。 HasFormContentType 属性用于确定表单是否已发布，并在 Razor 代码块中引用 Request.Form 集合，并将其中的相关值分配给 name 变量。

Razor 代码块由开头的 @{ 表示，并以结束的 } 结束。 块内的内容是标准的 C# 代码。

单个控制结构不需要代码块。 您可以简单地使用 @ 符号作为前缀。 前面示例中的 if 块对此进行了说明。

要呈现 C# 变量或表达式的值，请使用 @ 符号作为前缀，如 if 块中的 name 变量所示。

方法块

下一个示例产生与前一个示例相同的功能，但它使用 @functions 块来声明使用 BindProperty 属性修饰的公共属性，以确保 属性参与模型绑定，无需手动将表单值分配给变量。

Example.cshtml:

@page

@functions {
    [BindProperty]
    public string Name { get; set; }
}

<div style="margin-top:30px;">
    <form method="post">
        <div>Name: <input name="name" /></div>
        <div><input type="submit" /></div>
    </form>
    @if (!string.IsNullOrEmpty(Name))
    {
        <p>Hello @Name!</p>
    }
</div>

这种方法虽然是对前一种方法的改进，它利用了强类型，同时处理逻辑可以限制在 @functions 块中，但页面将变得更加难以维护和测试。 不过，可以在 @functions 块中声明方法甚至嵌套类。

您还可以使用 @functions 块来声明包含 HTML 的本地方法，以充当当前页面的显示助手。 如果页面有多个包含 HTML 的代码块并且需要应用类似的格式，则您可能会这样做。 这仅在 ASP.NET Core 3.x 中是可能的

在以下示例中，声明了一个格式化 DateTime 值的方法：

@functions{
    void DisplayDate(DateTime dt)
    {
        <span class="date">@dt.ToString("dddd, dd MMMM yyyy")</span>
    }
}

同时，假如您想在页面中的何处使用此格式呈现日期，都可以执行以下操作： 

@DisplayDate(myDate)

如果您需要修改输出，这将非常有用，因为您只需要在一处进行更改。

在 Razor Pages 2.x 中可以做类似的事情，但必须在代码块中声明该方法，如下所示： 

@{
    Func<DateTime, IHtmlContent> DisplayDate = @<span class="date">@item.ToString("dddd, dd MMMM yyyy")</span>;
}

item 变量是一个特殊的变量，它引用传递给方法的第一个参数。同样用法, 但您需要为其添加一个 using 引用 Microsoft.AspNetCore.Html

PageModel 文件

开发 Razor Pages 应用程序的推荐方法是将内容页面中的服务器端代码量降至最低。 任何与处理用户输入或数据相关的代码都应放在 PageModel 文件中，它们相互一一对应与其相关内容页面的映射。 它们甚至共享相同的文件名，尽管在末尾有一个额外的 .cs 来表示它们实际上是 C# 类文件的事实。

以下代码显示了适用于 PageModel 的 Example.cshtml 文件：

Example.cshtml:

@page
@model ExampleModel

<div style="margin-top:30px;">
    <form method="post">
        <div>Name: <input asp-for="Name" /></div>
        <div><input type="submit" /></div>
    </form>
    @if (!string.IsNullOrEmpty(Model.Name))
    {
        <p>Hello @Model.Name!</p>
    }
</div>

这是 PageModel 类的代码：

Example.cshtml.cs:

using Microsoft.AspNetCore.Mvc;
using Microsoft.AspNetCore.Mvc.RazorPages;

namespace RazorPages.Pages
{
    public class ExampleModel : PageModel
    {
        [BindProperty]
        public string Name { get; set; }
    }
}

PageModel 类有一个属性，如前一个示例中所定义，并使用 BindProperty 属性进行修饰。 内容页面不再有 @functions 块，但它现在包含一个 @model 指令，指定 ExampleModel 是 页。 这也会在页面中启用标签助手，进一步利用编译时类型检查。

默认项目生成与 PageModel 文件配对的内容页面。 这是推荐的方法。 但是，了解如何在不需要 PageModel 的情况下使用内容页面也很有用。
